# Tools And Prompt Contract

这份说明用于回答一个经常会混淆的问题：哪些东西应该写进运行配置，哪些东西应该写进 workspace bootstrap，哪些东西不能指望子智能体自动继承。

## 1. 最小共享合同

维护这套模板时，把下面两份文件当作主智能体和子智能体都应遵守的最小共享合同：

- `AGENTS.md`
- `TOOLS.md`

它们应该只放稳定、短小、跨任务都成立的规则。

## 2. 不要假设会自动继承的内容

以下内容不应被设计成“只要主智能体知道，子智能体自然也会知道”：

- 很长的背景说明
- 详细的技能流程
- 某次会话里刚写下的大段上下文
- 某条特定记忆的完整原文

如果子任务强依赖这些内容，派发时要显式说明。

## 3. 配置与文档的分工

改 `openclaw.json`：

- 启用或禁用工具
- memory backend / browser / plugin slot
- 模型、sandbox、timeout、compaction

改 `AGENTS.md`：

- 基础行为规则
- 写作和行动的最小共识

改 `TOOLS.md`：

- 工具选型顺序
- 常见任务的默认路由

改 `MEMORY.md`：

- 长期稳定事实
- 团队偏好
- 跨会话仍应保留的决策

改 `skills/`：

- 长工作流
- 案例
- 例外处理

## 4. 记忆写入边界

- `MEMORY.md`：稳定事实、稳定偏好、长期决策
- `memory/YYYY-MM-DD.md`：短期过程、阶段性结论、一次性上下文

不要把猜测、未经验证的结论或纯临时缓存写成长期记忆。

## 5. 委派子智能体时的建议

- 明确子任务的目标和边界
- 如需特定工具，直接写明
- 如需参考某个 skill 或某条记忆，直接点名
- 不要假设子智能体会自动读取完整长文档
